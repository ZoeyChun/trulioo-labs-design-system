01 — Buttons

Buttons

Trigger actions, submit forms, and navigate flows. Use one primary action per view — secondary and invisible variants for supporting actions.

Variants

Primary for the main CTA. Secondary for alternatives. Danger for destructive actions. Invisible for tertiary toolbar actions.

.tds-btn--{primary|secondary|danger|invisible} Required

Sizes

Small fits dense tables and chips. Medium is the default. Large for hero actions and full-width mobile CTAs.

.tds-btn--{sm|md|lg}

States

Disabled blocks interaction. Inactive shows a selected-but-muted filter state — text stays muted on hover.

.tds-btn--inactive · :disabled
Default Focused Disabled Inactive Primary Secondary Danger Invisible

03 — Controls

Controls

Binary and single-select choices — checkboxes, radios, switches, and radio cards. Entire rows should be clickable, not just the control.

Checkbox

Multi-select and boolean toggles. Support indeterminate for partial selections.

.tds-checkbox

Radio

Mutually exclusive options. Always group with a shared name attribute.

.tds-radio

Radio group

Stacked options with optional captions — ideal for verification type pickers.

.tds-radio-group

Switch sizes

Immediate on/off settings. Medium for forms; small for compact settings rows.

.tds-switch__track--{md|sm}
